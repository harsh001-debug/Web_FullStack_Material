# JavaScript — Structured Learning Notes

## 🎯 1. Introduction to JavaScript

### What is JavaScript?
JavaScript (JS) is a **programming language** used to add **logic and interactivity** to websites.

- Short form: **JS**
- Runs mainly in the browser
- Makes web pages dynamic

> ⚠️ **Important:** Java and JavaScript are completely different languages.

---

## 🧠 2. Why JavaScript is Needed

### Roles of Web Technologies

| Technology | Purpose |
|-----------|--------|
| HTML | Structure (body) |
| CSS | Styling (clothes) |
| JavaScript | Logic (brain) |

### Calculator Example

With only HTML & CSS:
- Inputs can be created ✅
- Button can be styled ✅
- **Cannot perform addition ❌**

With JavaScript:
- Takes user input
- Performs calculation
- Shows result

✅ **Conclusion:** JavaScript makes websites interactive.

## ⚡ 3. What JavaScript Can Do

### 3.2 Print Output

```js
console.log("Hello Coder Army");
```

**console.log()** prints output in the browser console.

### 3.3 Manipulate HTML (DOM)

Examples:
- Change text dynamically
- Show/hide elements
- Update content after button click

---

### 3.4 Manipulate CSS

Examples:
- Change background color on click

## ⚙️ 5. JavaScript Engine

### Definition
A **JavaScript engine** converts JavaScript code into machine code so the computer can execute it.

### Examples of JS Engines

| Browser | Engine |
|--------|--------|
| Google Chrome | V8 |
| Mozilla Firefox | SpiderMonkey |

---

### Execution Flow

```
JavaScript Code → JS Engine → Machine Code (0s & 1s) → Output
```

---

## 🖥️ 6. Why VS Code Cannot Run JS Directly

VS Code is only a **text editor**, not a runtime environment.

- Browser has JS engine ✅
- VS Code alone does not ❌

To run JS outside browser → we use **Node.js**.

---

## 🚀 7. Node.js — Key Idea

### Problem
Originally, JavaScript could run only inside browsers.

### Solution
**Node.js** allows JavaScript to run outside the browser.

### How Node.js Works

Node.js:
- Uses V8 engine
- Adds extra features
- Enables server-side JavaScript

---

### After Node.js

JavaScript can run:

- Frontend ✅
- Backend ✅
- Local system scripts ✅

This enabled the **MERN stack**.

## 🧩 9. Variables in JavaScript

JavaScript provides three keywords:

- **let** (modern variable)
- **const** (constant)
- **var** (old — avoid)

---

## ✅ 9.1 let (Recommended)

Used for **changeable values**.

```js
let num = 10;
console.log(num);
```

### Reassignment Allowed

```js
let name = "Rohit";
name = "Mohit"; // allowed
```

---

## 🔒 9.2 const (Constant)

Used for **fixed values**.

```js
const id = 20;
```

### Reassignment Not Allowed

```js
const id = 20;
id = 30; // ❌ Error
```

### When to Use const

Use for values that should never change:

- Bank account number
- User ID
- Configuration values

---

## ⚠️ 9.3 var (Old — Avoid)

```js
var x = 10;
```

### Status

- Old keyword
- Has scope issues
- Not recommended in modern JavaScript

✅ Prefer **let** and **const**.

---

## 🔄 10. Dynamic Typing (Important)

JavaScript is **dynamically typed**.

You do NOT specify data types manually.

```js
let num = 10;       // number
let name = "Rohit"; // string
let value = 10.5;    // float
```

JavaScript automatically detects the type.
